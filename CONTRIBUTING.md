# Contributing

Thanks for your interest in improving this project.

This project started as a practical tool for Infor ION-related development, but the hope is for it to be adapted, improved, and expanded by others.

## Ways You Can Contribute

You can contribute by:

- Reporting bugs
- Suggesting enhancements
- Improving documentation
- Adding examples
- Testing with real-world ION payloads or workflows
- Submitting code changes

## Before You Start

Please check the existing Issues first.

Someone may have already reported the bug or suggested the same enhancement.

## Reporting Bugs

When reporting a bug, please include:

- What you were trying to do
- What happened
- What you expected to happen
- Any error messages
- Sample input or payload, if possible
- Your Python version
- Your operating system

Please remove or mask any sensitive customer, tenant, credential, or production data before posting examples.

## Suggesting Enhancements

Enhancement ideas are welcome.

A good enhancement request should explain:

- The problem you are trying to solve
- Why the change would be useful
- What the desired behavior should look like

## Submitting Changes

If you want to make code changes:

1. Fork the repository
2. Create a new branch
3. Make your changes
4. Test your changes
5. Submit a pull request

Example:

```bash
git checkout -b feature/my-improvement
Pull Request Guidelines
```

## Before submitting a pull request:

- Keep the change focused
- Include a clear description
- Add or update documentation if needed
- Test the change locally
- Avoid including customer-specific data or credentials
- Coding Guidelines

## Please keep the project:

- Simple
- Readable
- Dependency-light
- Friendly to Infor ION scripting environments

Avoid adding external dependencies unless there is a strong reason.

## Sensitive Data Warning

## Do not commit:

- .ionapi files
- API keys
- Secrets
- Customer payloads
- Tenant names
- Production data
- Personally identifiable information

Use sanitized sample data instead.

## Good First Issues

- Issues labeled good first issue are intended to be smaller, approachable tasks for new contributors.

- These are a good place to start if you want to help but are not sure where to begin.

## Project Direction

This project is meant to be a useful starting point.

Fork it, customize it, improve it, and make it work for your environment.
